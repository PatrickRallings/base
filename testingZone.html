ARPANET (Advanced Research Projects Agency Network)
    • Initial stages of the internet which started ni the late 1960's and early 1970's
    • Government project which sought out the ability for computers to communicate with one another

    The Wired Internet
    • Early stages of the internet utilized existing telephone lines
    • Current iteration of the internet
    • Internet is broken up into subsections/local and regional networks which then connect to backbone networks
    • Backbone
      • The main base for internet communications
    • Backbone -> Regional networks -> Local networks -> Users
    • Routers are used to determine which packets of information should be going to which sources
      • Without routers, computers would have to be connected to each other on a 1 to 1 basis (such as a network of 10 computers requiring 9 cable ports per computer with a network total of 45 different cables)
    • Modems convert information from our routers/networks into types of information that is manageable by telephone or other infrastructure
    • ISP, or Internet Service Providers, essentially have very powerful routers that route information where it is supposed to go on a massive scale

    Addresses (Subnetting)
    • IP Addresses
      • Specific address for your network (or your computer, depending on your router setup)
        • IP addresses are given through a process called subnetting
      • Example (IP v4): 123.45.678.9
        • IP v6 is a newer type of IP which utilizes the same standard as above but which additional decimals/hexadecimals
      • Visually looks pretty similar for most routers (1.192.**.***.*) but has a more specific address in the underneath which is constantly being scrambled
    • MAC Addresses
      • Addresses specific to an exact device which never changes and is only shared between trusted devices
      • IP address sits on top of this MAC address
    • Domain (URL, or Uniform Resource Locator)
      • Location of server that end-users will connect to through their IP
    • DNS (Domain Name Server)
      • Servers that essentially contain a large "phonebook" which contains records of all of the domains in their purview
      • They use their domain location information to connect IPs to their various domains
      • They license out domain names
      • ".com"
        • They most common ending in a URL
        • Regulated by the government
      • ".gov"
        • Only government institutions can end their domains in this
      • ".org"
        • Typically for non-profit organizations and larger institutions
      • ".edu"
        • Specifically for educational bodies
    • Order of Addressing
      • User types in URL
      • DNS receives request
      • DNS returns actual IP address for URL back to user
      • User uses actual IP to locate website
      • Website receives user's IP; at this point both end points have Ips and thus can communicate further without help from DNS
    • HTTP (Hypertext Transfer Protocol)
      • An application protocol that defines a language for clients and servers to talk to one another

    Webpages
    • Websites are essentially just files down to their barebones
    • UNIX (Apache Web Server)
      • Server that hosts all of the main files for a website
      • Larger websites may not have all of the files on this initial server and will have to pull more files from an Oracle Database
    • HTML (Hyper Text Markup Language)
      • The language used to structure and present the text that we usually see within webpages (front-end framework of code)
      • Purely structures the content on a webpage
      • Runs anything that is interacted with on a webpage
      • Uses CSS to stylize the HTML structure
      • HTML has been through several iterations and is currently on HTML5
    • CSS (Cascading Style Sheets)
      • Formats the HTML structure visually
      • Allows the content in a webpage to remain the same regardless of style needs
        • Example: JavaScript can tell CSS what kind of device, such as a phone or pc, is viewing the website, and allow the client to view the same content as other devices but with CSS code specific to their device's needs
    • JavaScript (JS)
      • Works with HTML and CSS to provide the actual "motion" for different interactions on a website
      • 97% of websites use JavaScript
      • Example: The code behind the action in clicking a button would usually be written in JavaScript
    • Component files
      • Code files: Files which host the actual code for websites, such as code written in HTML
      • Assets: Files which hold the various assets that make up a website, which would otherwise be very complex to have inside the base code
        • Example: Images, word documents, etc.

    Information Systems
    • Cloud Computing
      • The use of software and computing processes that take place outside the computer you are on (the software isn't even installed on the client's computer)
      • Examples: iCloud, Google Drive, etc. (even email)
        • Most consumer emails, such as Gmail, use cloud computing by storing the account's information and email software externally rather than on the client's device
    • Private Cloud
      • A scaled down version of normal cloud computing where the software computing processes take place outside of a client's computer but still within their local network
      • This tool is being utilized more and more as organizations look for efficiency and security
    • Virtualization
      • A form of cloud computing where entire computers can be simulated on another computer, usually doing tasks that would not normally be possible on the client computer
